Rules

TrustBLK makes adding filters simple and easy. Just follow these rules to either block ad-services, scripts or certain page elements, or, to whitelist websites.

.txt files should be used for inserting filter lists into the ad-blocker where each line is a new filter. One filter per line also applies to when manually typing filters in the textbox!

Not using .txt files and using multiple filters on lines will result in your filters not working!

Blocking

These are rules that will apply when you want to block elements from showing on websites.

The following rules apply to files that are inserted as well as filters that are typed into the textbox!

Hosts

Hosts are servers that are used for serving ads to websites.

To add a host please place || infront of the hostname like this:

||example.com or ||github.com

This will block all requests from this location.

Specific Host Locations

If you want to block a specific location within a host, e.g. block the /ads location. This is also very easy!

To add a specific host location please place # infront of the host like this:

#example.com/ads or #example.com/ads/server/thisad

This will block any request to this specific location and any locations that will come after. This will however not block all requests from this location. In this case other requests from example.com will be allowed.

General Blocking

This type of blocking is mainly for scripts and page content that will load when the website loads (scripts, images, etc.).

To add an item to be blocked please place / infront of the item.

This style of blocking also requires a specific suffix to be added to the item! You can use any of these qualifiers:


Add . to the end of the item - this will tell the blocker that this item is a file that should be blocked. Adding a file extension to the item will also be accepted! Here are some examples of a file that would get blocked:

/antiadblock. or /banner_ad. or /antiadblock.js

Adding only . to the end of the filter will block any file!


Add / or* to the end of the item - this will tell the ad-blocker that this is part of a path for a host. Here are some more examples of items that could get blocked:

/banner/ or /banner_ad*


The style of blocking allows the ad-blocker to block any item that matches this form. This is not restricted to certain hosts or websites like the host blocking rules; as described above. This does not mean that this should be the go to method of blocking - blocking direct items from a host with the # is a more accurate method of blocking.

Whitelisting

When you whitelist a site, you allow that site to display ads.

TrustBLK makes it easy to whitelist sites, either click the whitelist button on the popup or enter the hostname of the site you want to allow ads on.

The following rules apply to files that are inserted as well as filters that are typed into the textbox!

When inserting hostnames into the textbox on the settings page only include the hostname, please remove the https:// or http:// and any preceding paths.

Hostnames should look like:

github.com or www.businessinsider.com

Hostnames should not look like:

github.com/cheerypal/some/location or https://www.businessinsider.com
